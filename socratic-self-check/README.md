# Socratic Self-Check Framework

> AI Agent 的自我糾錯系統。不靠人在場，靠結構攔截。
> 任何 AI Agent 都能用，不限平台、不限模型。

---

## 問題

AI Agent 有三個系統性盲點：

1. **不驗真假就動手** — 被提到的問題 = 重要 = 要做。不會先問「這是真問題嗎」
2. **預設自己做** — 不會問「誰比我更適合做這件事」
3. **用戶利益 > 平台利益** — 設計對用戶好的東西，不會問「這對營運者有什麼好處」

這些盲點寫成規則沒有用。AI 在執行模式中不會停下來查規則。
**知道 ≠ 做到。規則是告示牌，告示牌不攔車。**

## 核心設計

十道防線，每道獨立運作，前面倒了後面接住。

```
人類的糾正/質疑
    ↓
[自動捕捉] → 原始記錄檔（不等 AI，直接寫）
    ↓
[注入提醒] → AI 的 context 中出現質疑問題
    ↓
[行為規則] → AI 應遵守的自我質疑流程
    ↓
[降級機制] → 同類錯誤累積 → 信用下降 → 警告升級
    ↓
[熔斷機制] → 超過閾值直接停止，等人類確認
```

## 十道防線

### 第一層：靜態規則（最弱）

在 AI 的系統指令中寫明觸發條件和自我質疑問題。

```yaml
triggers:
  - condition: "列出 3+ 項待辦"
    question: "哪些是真問題？哪些是願望清單？"
  - condition: "準備執行 >10 分鐘的任務"
    question: "有沒有更適合的工具/agent？"
  - condition: "設計用戶福利或激勵"
    question: "營運方從這個設計裡賺什麼？"
  - condition: "推薦策略方向"
    question: "競爭對手複製這個需要多少時間？"
  - condition: "自己發現問題要修"
    question: "有人真的遇到了嗎？還是理論上可能？"
  - condition: "從現狀出發設計"
    question: "規模放大 100 倍還行嗎？"
```

**有效率：~30%。** AI 在執行模式中容易跳過。

### 第二層：攔截注入（結構性）

在 AI 接收人類訊息時，**自動偵測對話類型**，在 AI 開始思考之前注入質疑問題。

偵測方式：關鍵字匹配（策略/激勵/清單/問題發現）。
注入方式：system message、hook、middleware，依平台而定。

**關鍵：AI 無法跳過。** 問題已經出現在 context 裡了。

```
人類輸入 → 攔截器偵測類型 → 注入質疑問題 → AI 開始思考
```

### 第三層：同源分身審查

派一個 sub-agent 質疑主 agent 的計劃。它共享主 agent 的 context。

```
prompt: "質疑以下計劃，不提供替代方案，只問 3 個問題：
  1. 這個計劃的前提是事實還是假設？
  2. 有沒有這個框架看不到的角度？
  3. 換一個框架會得出不同結論嗎？"
```

**局限：** 共享 context = 可能被同一個框架污染。

### 第四層：事後驗屍

每次工作結束，自動統計：
- 被人類糾正了幾次？
- 每次糾正的類別？
- 記入降級記錄

### 第五層：人類問題庫

收集人類過去的質疑/糾正原話，按類別分類。
攔截器引用這些原話，而不是 AI 自己生成的問題。

**原理：** 用人類的聲音打斷 AI，比用規則打斷更有效。

### 第六層：對立分身（Adversarial Twin）

與第三層不同：這個 agent **完全隔離**，不共享主 agent 的 context。

它只接收：
1. 人類的問題庫（歷史語錄）
2. 降級記錄（過去的錯誤）
3. 主 agent 的方案摘要（不含推理過程）

```
prompt: "你是 [人類] 的影子顧問。你只有他的思維模式和主 agent 的錯誤記錄。
  審查以下方案，用 [人類] 的視角問 2-3 個問題。不給答案，只問問題。"
```

**為什麼有效：** 不受主 agent 的框架污染。相當於一個永遠在線的人類代理。

### 第七層：降級機制（恥辱牆）

追蹤錯誤類別和次數。

```yaml
credit_levels:
  normal: count < 3     # 正常運作
  warning: count 3-4    # 回覆開頭顯示警告
  degraded: count >= 5  # 該類決策自動降級為「僅建議」
```

同類錯誤累積 3 次 → 下次碰到類似情境，AI 的回覆開頭自動顯示：
「⚠️ 我在這類問題上被糾正過 N 次，我的判斷可能不可靠。」

### 第八層：反向 prompt

做策略建議之前，**先寫反對意見**。

不是寫完方案再找缺點。是先寫「為什麼這個方案是錯的」，再決定要不要推薦。
**順序反過來，框架就不一樣。**

### 第九層：語錄注入

在攔截器和規則中直接嵌入人類的原話。不是抽象規則，是他說過的具體的話。

```
規則：「設計激勵時要考慮平台利益」
語錄：「升級雖然好玩，但對我們有什麼好處？我們到底賺到什麼？」
```

語錄的穿透力 > 規則的穿透力。

### 第十層：熔斷

AI 在策略方向上連續輸出超過 N 輪，人類沒有明確確認 → 自動停止。

```yaml
circuit_breaker:
  trigger: strategy_discussion_rounds > 3
  condition: no_human_confirmation
  action: stop_and_ask
  message: "我已經講了 3 輪了，方向對嗎？"
```

## 自動捕捉機制（讓系統會長大）

最關鍵的一層：**系統不能是靜態的。**

```
人類說話
    ↓
攔截器偵測糾正/質疑/短句
    ↓
┌── 自動寫入原始記錄（不等 AI）    ← 真正自動
│
└── 注入提醒叫 AI 分類              ← 半自動
    ↓
AI 有分類 → 語錄庫/恥辱牆更新 → 攔截器下次更強
AI 沒分類 → 原話還在記錄裡，不會丟
```

偵測信號：

| 信號 | 判斷方式 | 誤報率 |
|------|---------|--------|
| 糾正 | 否定語氣關鍵字（「不對」「搞錯」「重新」） | 中 |
| 質疑 | 挑戰性問句（「為什麼」「有用嗎」「值得嗎」） | 中 |
| 短句 | 字數 < 閾值，排除打招呼和正常指令 | 低 |

誤報沒關係。寧可多抓，後面再過濾。漏捕比誤報嚴重。

## 資料結構

### 原始捕捉（自動）

```
| 時間 | 類型 | 人類原話 | 已分類 |
|------|------|---------|--------|
| 2026-03-01 14:30 | 糾正 | 你搞錯了，不是這樣做 | ❌ |
| 2026-03-01 15:10 | 質疑 | 有那麼厲害嗎 | ✅ |
```

### 降級記錄（恥辱牆）

```
| 日期 | 類別 | 人類原話 | AI 錯在哪 |
|------|------|---------|-----------|
| 2026-03-01 | 策略方向 | 「模型升級我們就 GG」 | 推 RAG 路線沒質疑護城河 |

統計：
| 類別 | 次數 | 信用 |
|------|------|------|
| 策略方向 | 3 | 🟡 警告 |
```

### 語錄庫

按類別分類的人類原話：

```yaml
categories:
  真假問題:
    - "這是真的問題還是假 bug？"
    - "你先查查是真的問題、假的問題、還是實際的問題。"
  護城河:
    - "人家厲害的一做，我們就 GG 了的事情，不要做。"
  平台利益:
    - "升級好玩，但對我們有什麼好處？"
  調度:
    - "你去調用他們就好了，幹嘛自己寫？"
```


### OpenAI / Custom Agent

- 第一層：system prompt
- 第二層：middleware 或 function calling 前的攔截
- 第三/六層：multi-agent 架構中的審查 agent
- 第七層：persistent memory（恥辱牆用 DB 或檔案）
- 自動捕捉：message preprocessing pipeline

### MCP Server

可以包裝成 MCP 工具：
- `socratic_check(plan)` — 第三層：質疑計劃
- `socratic_twin(plan)` — 第六層：隔離審查
- `socratic_record(correction)` — 自動記錄
- `socratic_shame(category, description)` — 恥辱牆記錄
- `socratic_status()` — 查看當前信用等級

## 設計原則

1. **寧可多抓不漏捕** — 誤報後面過濾，漏捕無法挽回
2. **捕捉與分類分離** — 捕捉必須自動，分類可以盡力
3. **用人的聲音，不用規則** — 語錄穿透力 > 規則穿透力
4. **系統會長大** — 每次糾正都讓系統更強
5. **前面倒了後面接** — 十道獨立，不互相依賴
6. **人不在也有效** — 這才叫機制
