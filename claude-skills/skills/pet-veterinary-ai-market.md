# 寵物 AI 獸醫市場分析

## 核心定位（業主指示）

> 「寵物 AI 這件事情，我們要用心去做。」
>
> 不是取代獸醫，而是：
> 1. 幫助飼主在第一時間做正確判斷
> 2. 讓飼主知道怎麼跟獸醫有效溝通
> 3. 獸醫很遠、看診很貴 → AI 先幫忙篩選和準備

## 市場概況

| 項目 | 數據 |
|------|------|
| **全球寵物醫療 AI 市場** | 預估 2026 年 $2.5B+ |
| **日本寵物飼養數** | 犬 684 萬頭 + 貓 906 萬頭（2024） |
| **日本年均獸醫費用** | 犬 ¥67,000 / 貓 ¥35,000 |
| **飼主第一反應** | 80%+ 先上網搜尋症狀 |

## 主要競品分析

### 1. もしぺ（Moshi Pe）— 日本市場領先者

| 項目 | 詳情 |
|------|------|
| **開發者** | Anipos（日本保險科技公司） |
| **使用次數** | 671,000+ |
| **價格** | **免費** |
| **介面** | **LINE 整合**（日本用戶最自然的入口） |
| **功能** | 症狀輸入 → AI 判斷 → 建議是否就醫 |
| **特色** | 有獸醫監修、與保險連動 |
| **弱點** | 只做症狀對照，無照片分析、無行為分析 |

### 2. TTcare — 韓國技術領先

| 項目 | 詳情 |
|------|------|
| **來源** | 韓國 AI 新創 |
| **準確率** | **95%**（眼部、皮膚照片診斷） |
| **功能** | 照片上傳 → AI 分析眼部/皮膚疾病 |
| **特色** | 多模態 AI、醫學影像等級 |
| **弱點** | 限定眼部/皮膚，無全身評估 |

### 3. 藤原動物病院 — ChatGPT 先驅

| 項目 | 詳情 |
|------|------|
| **位置** | 日本（具體診所） |
| **特色** | 日本首間導入 ChatGPT 輔助問診的獸醫院 |
| **模式** | 獸醫端工具（非消費者端） |
| **意義** | 證明了 AI 在獸醫領域的可行性 |

## 藍海領域（競品未覆蓋）

| 領域 | 現狀 | 機會 |
|------|------|------|
| **急診分診** | 多數只說「去看醫生」 | AI 精準判斷：緊急/明天看/觀察 |
| **行為分析** | 幾乎無人做 | 從行為變化判斷早期疾病 |
| **老年寵物照護** | 嚴重不足 | 犬貓 CDS、關節炎等老年病 |
| **營養個人化** | 通用建議 | 依品種/年齡/病史的飲食計劃 |
| **獸醫溝通準備** | 無 | 幫飼主整理症狀、準備問題清單 |
| **多語言** | 幾乎全日文/全英文 | 中日雙語 → 在日台灣人市場 |

## 技術架構方向

```
飼主輸入（LINE / Web）
    ↓
症狀描述 + 照片（可選）
    ↓
AI 分析引擎
├─ NLP：症狀理解與分類
├─ Vision：照片分析（皮膚/眼/口腔）
├─ Knowledge Graph：獸醫知識庫
└─ Risk Scoring：緊急度評分
    ↓
輸出
├─ 初步判斷（緊急/非緊急/觀察）
├─ 可能原因（簡明解釋）
├─ 給獸醫的問診準備清單
└─ 附近獸醫院推薦（可選）
```

## 業主的明確指示

1. **「用心去做」** — 這不是隨便做的副業，是核心方向
2. **「確認資料正確性」** — 醫療資訊的準確性是第一優先
3. **「幫助飼主第一時間判斷」** — 定位是急救/分診，不是替代獸醫
4. **「讓飼主知道怎麼跟獸醫溝通」** — 橋樑角色
5. **「討論清楚再開工」** — 不急著寫代碼，先把方向想清楚

## Notes

- 醫療 AI 需要獸醫專業監修（不能純 AI 生成內容）
- 日本需要確認是否涉及「獣医師法」的規範（AI 能做的 vs 不能做的）
- LINE 是日本最佳入口（月活 9,400 萬用戶）
- もしぺ 的免費模式值得研究 — 可能靠保險導流獲利
- 此方向需要與「動物照護」領域知識積累同步進行
