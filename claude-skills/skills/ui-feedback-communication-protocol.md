# UI 反饋高效溝通協議

## Problem
非技術人員想反饋 UI 問題時：
- 不知道如何描述位置（「那個按鈕」「上面那個東西」）
- 一個個反饋效率低（改一個 → 看一個 → 再改一個）
- 遠端協作無法當面指著螢幕說
- 開發者需要反覆確認「你說的是哪裡？」

## Context / Trigger Conditions
- 用戶說「我想反饋一些 UI 問題」
- 用戶說「這裡想改一下」但描述不清楚
- 進入 UI Review / QA 階段
- 需要建立長期的反饋流程

## Solution

### 方式一：文字模板（最簡單）

```markdown
📍 頁面：/pet/profile/chat
❌ 問題：按鈕太小，手機上很難點
✅ 期望：改大一點，至少 44px

---

📍 頁面：/explore
❌ 問題：頭像排列太擠
✅ 期望：增加間距，或改成 2 列顯示
```

**關鍵要素**
- 📍 **頁面路徑**：從網址列複製
- ❌ **問題描述**：現在是什麼樣
- ✅ **期望結果**：想要變成什麼樣

### 方式二：截圖標註（推薦）

**手機操作**
1. 截圖
2. 用內建編輯功能畫圈/箭頭
3. 直接發給開發者
4. 附一句說明

**優點**：直觀、快速、不需要安裝工具

### 方式三：專業工具（長期協作）

| 工具 | 特點 | 適用場景 |
|------|------|---------|
| **Markup.io** | 直接在網頁上畫圈標註 | 需要精確指出位置 |
| **Loom** | 錄影邊操作邊說明 | 複雜流程問題 |
| **Figma** | 在設計稿上評論 | 有設計稿的專案 |
| **Pastel** | 網頁標註 + 自動追蹤 | 專業 QA 團隊 |

**Markup.io 使用流程**
```
1. 安裝 Chrome 插件：https://markup.io
2. 打開要反饋的網站
3. 點擊插件 → 畫圈/箭頭/寫字
4. 產生分享連結發給開發者
```

### 方式四：批量收集（高效率）

**收集階段**
```markdown
🔴 P0 必修（影響使用）：
1. /login - 按鈕點不到
2. /cart - 價格顯示錯誤

🟡 P1 建議（體驗優化）：
3. /explore - 想換個顏色
4. /profile - 想加個按鈕

🟢 P2 nice-to-have：
5. /settings - icon 可以更可愛
```

**流程**
1. 業主花 30 分鐘完整逛一遍
2. 把所有問題記錄下來
3. 一次性發給開發者
4. 開發者批量修改
5. 業主再驗收

**為什麼這樣更快？**
- 開發者可以規劃修改順序
- 避免頻繁切換上下文
- 減少溝通來回次數
- 一次 `npm run build` 驗證所有修改

## Verification
- [ ] 業主能清楚表達問題位置
- [ ] 開發者一次就能理解需求
- [ ] 修改後業主確認「對，就是這樣」

## Example

### 好的反饋範例
```
📍 頁面：/pet/profile/chat/chat
❌ 問題：輸入框在 iPhone 上會被鍵盤擋住
✅ 期望：鍵盤彈出時輸入框要跟著往上移
📱 測試設備：iPhone 15 Pro, iOS 18
```

### 不好的反饋範例
```
那個聊天的地方有問題，打字的時候怪怪的
```
（缺少：哪個頁面？什麼問題？什麼設備？期望什麼結果？）

### 批量反饋範例
```
## 2026-01-29 UI Review 第一輪

### 🔴 P0 必修
1. `/login` - LINE 登入按鈕在 Android 上顏色不對
2. `/cart` - 總價計算好像少了稅金

### 🟡 P1 建議
3. `/explore` - 熱門動物那排太擠，希望改成可以左右滑動
4. `/pet/profile/chat` - 追蹤按鈕希望更明顯一點

### 🟢 P2 之後再說
5. `/settings` - icon 風格想統一
```

## Notes

### 給業主的建議
1. **截圖勝過千言萬語**：畫個圈比寫一段話清楚
2. **說清楚「期望結果」**：不只說哪裡不好，要說想要什麼樣
3. **批量收集**：花 30 分鐘逛完全站，一次發完比零散發好

### 給開發者的建議
1. **主動提供模板**：告訴業主用什麼格式反饋
2. **確認理解**：收到後用自己的話複述一次
3. **批量處理**：收集完再一起改，不要邊收邊改

### 工具選擇指南
| 情境 | 推薦工具 |
|------|---------|
| 快速一次性反饋 | 截圖 + 手機編輯 |
| 長期專案協作 | Markup.io |
| 複雜操作問題 | Loom 錄影 |
| 有設計稿 | Figma 評論 |
