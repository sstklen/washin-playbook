# 🔐 網路安全（給非工程師看的版本）

> **你不用懂這些技術名詞，Claude 會自動幫你處理**

---

## 🎯 用日常話解釋「安全問題」

### 密碼/API Key 寫死在代碼裡

**像什麼？**
> 把你的信用卡號寫在門口的白板上

**會發生什麼？**
> 任何人看到你的代碼就能用你的帳號、花你的錢

**Claude 會怎麼做？**
> 自動把密碼移到安全的地方（環境變數），不會出現在代碼裡

---

### SQL Injection（資料庫注入）

**像什麼？**
> 壞人在登入表格輸入魔法咒語，就能偷看/刪除你所有資料

**會發生什麼？**
> 你的用戶名單、訂單、私密資料全部被偷走或刪掉

**Claude 會怎麼做？**
> 把「危險的寫法」改成「安全的寫法」（參數化查詢）

---

### XSS 攻擊（跨站腳本）

**像什麼？**
> 壞人在你的網站留言板貼病毒，其他人看到就中毒

**會發生什麼？**
> 用戶的帳號被偷、被導到假網站、電腦被控制

**Claude 會怎麼做？**
> 過濾所有用戶輸入的內容，不讓病毒代碼執行

---

### 個資外洩

**像什麼？**
> 用戶的名字、Email、電話、地址被壞人看到

**會發生什麼？**
> 用戶被騷擾、被詐騙、告你違反個資法

**Claude 會怎麼做？**
> 加密儲存、限制誰能看、不存不必要的資料

---

### 沒有 HTTPS

**像什麼？**
> 寄明信片而不是密封的信，路上誰都能看

**會發生什麼？**
> 用戶輸入的密碼、信用卡號在傳輸過程被攔截

**Claude 會怎麼做？**
> 強制使用 HTTPS（加密傳輸）

## 📋 安全檢查清單（用日常話）

### 上線前必須確認

- [ ] **密碼和 Key 有藏好嗎？**
  - 不能直接寫在代碼裡
  - 要放在 .env 檔案
  - .env 不能上傳到 GitHub

- [ ] **用戶輸入有檢查嗎？**
  - 名字、Email、留言等都要過濾
  - 不能直接拿去查資料庫

- [ ] **傳輸有加密嗎？**
  - 網址要是 https:// 開頭
  - 不能是 http://

- [ ] **誰能看什麼有限制嗎？**
  - 用戶只能看自己的資料
  - 管理員才能看全部

- [ ] **個資有保護嗎？**
  - 密碼要加密儲存（雜湊）
  - 不存不必要的資料

## 📚 常見問題

### Q: 什麼是環境變數？

**日常解釋：**
> 一個只有你的電腦/伺服器知道的秘密小抄。
> 代碼裡只寫「去看小抄」，不寫實際內容。
> 這樣即使代碼被看到，密碼還是安全的。

### Q: 什麼是 .env 檔案？

**日常解釋：**
> 存放秘密小抄的檔案。
> 格式像這樣：
> ```
> API_KEY=你的密鑰
> PASSWORD=你的密碼
> ```
> 這個檔案絕對不能上傳到 GitHub。

### Q: 什麼是 .gitignore？

**日常解釋：**
> 告訴 Git「這些檔案不要上傳」的清單。
> 把 .env 加進去，密碼就不會被上傳。

### Q: 為什麼 HTTP 不安全？

**日常解釋：**
> HTTP = 寄明信片，路上誰都能看
> HTTPS = 寄密封的信，只有收件人能看
>
> 用戶輸入密碼時，如果是 HTTP，
> 中間的任何人（咖啡店 WiFi、網路商）都能看到。
